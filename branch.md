&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Integração Contínua (CI, da sigla em inglês) é uma prática do DevOps que possui duas dimensões: a primeira, os desenvolvedores commitam e
dão merge em suas branches com frequência, tendo pequenas alterações sendo adicionadas a cada modificação, ao invés de grandes alterações 
de uma só vez; a segunda, após os commits e merges serem feitos, um ambiente de testes automatizado executa os referidos testes na versao
modificada do código.  
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;  Branches (ramificações do código principal) são importantes para que cada desenvolvedor faça suas atividades no código de modo indepente,
sem conflitar com as alterações dos demais desenvolvedores, com posterior integração ao código através de merges.  
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;  O pull-request é revelante é-lhe devido a integração das várias branches com o código principal (normalmente, na branch "main"). O pull-
-request permite revisão das modificações antes da integração.
